Project Overview

This project explored how weak passwords can create security risk in network services such as SSH and FTP. The goal was not only to understand authentication testing concepts, but also to learn how organizations can defend against password-based attacks.

Ethical Lab Notice

This project was performed only in an authorized training lab environment. The purpose was learning, documentation, risk understanding, and defensive improvement.

Methodology

My process.

01

Lab Preparation

I prepared a safe practice environment and worked only inside the authorized lab network.

02

Authentication Concepts

I studied how password-auditing tools test username and password combinations against SSH and FTP.

03

Access Verification

I verified the concept in the controlled environment and documented the defender’s perspective.

04

Defensive Thinking

I focused on strong passwords, lockout policies, monitoring, and disabling unnecessary services.

What I Learned

Weak credentials can expose important services. Even basic services like SSH and FTP require strong access controls.

Recommended Mitigations

  • Use strong and unique passwords.
  • Enable multi-factor authentication.
  • Limit failed login attempts.
  • Monitor authentication logs.
  • Use key-based SSH authentication.
