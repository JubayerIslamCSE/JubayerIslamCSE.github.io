Project Overview

This project explored how weak passwords can expose network services such as SSH and FTP. The goal was to understand the concept in a safe lab and think about mitigation from a defender’s perspective.

Ethical Notice

This project was performed only in an authorized training lab environment. It is documented for learning, portfolio presentation, and defensive awareness.

Methodology

My process.

01

Lab Preparation

I prepared a safe environment and worked only in the authorized lab network.

02

Authentication Concepts

I studied how password-auditing tools work against SSH and FTP authentication.

03

Verification

I verified the concept and documented the implications from a security standpoint.

04

Defensive Thinking

I focused on strong passwords, lockout policies, monitoring, and disabling unnecessary services.

What I Learned

Weak credentials can expose important services. Strong authentication and access-control practices are essential.

Recommended Mitigations

  • Use strong and unique passwords
  • Enable multi-factor authentication where possible
  • Limit failed login attempts
  • Monitor authentication logs
  • Use key-based SSH authentication where applicable
